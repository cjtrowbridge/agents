# Assimilation Trail

This directory preserves the framework's learning history from external agentic frameworks and adjacent projects.

## Purpose

Record:
- what was studied,
- what was adopted,
- what was rejected or deferred,
- and why.

This prevents repeated re-analysis and makes future comparisons cumulative instead of repetitive.

## How to Use

1. Run the assimilation workflow in `./playbooks/how_to_assimilate_another_agentic_framework.md`.
2. After approved changes are implemented (or after a review-only round), create a new entry in this directory.
3. Use `./templates/assimilation_trail_entry.md` as the starting point.
4. Include concrete file references for adopted changes when possible.

## Naming

Use date-first filenames for easy sorting:

- `YYYY-MM-DD-framework-name.md`

Example:

- `2026-02-24-get-shit-done.md`
